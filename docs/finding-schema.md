# Finding Schema Reference

Kulshan emits **findings** — structured audit observations produced by each pack. Every finding conforms to a canonical schema defined in `models.py`. This page is the single-source reference for that shape.

## Required Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique finding identifier. Generated by `make_finding_id()`. |
| `pack` | string | The audit pack that produced this finding. |
| `kind` | string | Finding sub-type within the pack (e.g. `unused-ebs-volume`). |
| `title` | string | Human-readable one-liner describing the finding. |
| `severity` | enum | Impact level. |
| `confidence` | float | How certain the detector is (0.0–1.0). |
| `effort` | enum | Estimated remediation effort. |
| `risk` | enum | Risk of acting on the recommendation. |

## Field Details

### `id`

Format: `{pack}-{kind}-{hash}` — generated via `make_finding_id()`. The hash suffix ensures uniqueness across resources. Max 512 chars, non-empty.

### `pack`

Valid values:

| Pack | Domain |
|------|--------|
| `cost` | Cost analysis and anomaly detection |
| `security` | IAM, network, encryption posture |
| `sweep` | Orphaned resource detection |
| `dr` | Disaster recovery posture |
| `age` | Lifecycle and staleness audit |
| `drift` | CloudFormation drift and IaC coverage |
| `tag` | Tag compliance |
| `pulse` | Observability and alarm coverage |
| `limit` | Service quota headroom |
| `topo` | VPC topology and route integrity |

### `kind`

Pack-specific string identifying the check type (e.g. `unused-ebs-volume`, `public-s3-bucket`). Non-empty, max 512 chars.

### `title`

Human-readable summary. Non-empty, max 512 chars.

### `severity`

| Value | Meaning |
|-------|---------|
| `critical` | Immediate action required — active risk or major waste |
| `high` | Should be addressed soon |
| `medium` | Worth investigating |
| `low` | Minor concern |
| `info` | Informational, no action needed |

### `confidence`

Float or int in the range `[0.0, 1.0]`. Represents detector certainty. A value of `1.0` means deterministic (e.g. resource confirmed unused via CloudTrail). Lower values indicate heuristic or statistical inference.

### `effort`

Estimated time to remediate:

| Value | Time |
|-------|------|
| `trivial` | 0–15 minutes |
| `low` | 16–60 minutes |
| `medium` | 61–240 minutes |
| `high` | 241+ minutes |

### `risk`

Risk of acting on the recommendation (i.e. deleting the resource, changing the config):

| Value | Meaning |
|-------|---------|
| `safe` | No production impact expected |
| `low` | Minimal risk, easily reversible |
| `medium` | Some blast radius, verify first |
| `high` | Could affect production, requires planning |

## Optional Fields

| Field | Type | Description |
|-------|------|-------------|
| `description` | string | Extended explanation of the finding. |
| `resource_arn` | string | Full ARN of the affected resource. |
| `resource_id` | string | Short resource identifier (e.g. `vol-0abc123`). |
| `region` | string | AWS region where the resource lives. |
| `recommendation` | string | What to do about it. |
| `savings_monthly` | float | Estimated monthly savings in USD (cost pack). |
| `metadata` | dict | Arbitrary key-value pairs for pack-specific context. |
| `remediation` | dict | Structured remediation steps (action, service, params). |
| `compliance` | list | Relevant compliance frameworks (e.g. `["CIS 1.4", "SOC2"]`). |

## Validation Rules

A finding is **invalid** if any of the following are true:

- Any required field is missing or `null`.
- A string field (`id`, `pack`, `kind`, `title`) is empty or exceeds 512 characters.
- `pack` is not one of the 10 valid pack names.
- `severity` is not one of: `critical`, `high`, `medium`, `low`, `info`.
- `confidence` is outside the range `[0.0, 1.0]`.
- `effort` is not one of: `trivial`, `low`, `medium`, `high`.
- `risk` is not one of: `safe`, `low`, `medium`, `high`.

Invalid findings are rejected at emission time and logged as collector errors.

## Fingerprint

Each finding includes a SHA256 fingerprint computed from stable fields (`pack`, `kind`, `resource_arn` or `resource_id`, and key metadata). This enables deduplication across runs — if the same issue appears in consecutive audits, the fingerprint stays constant.

## Legacy Format (v1 → v2)

Schema v1.0 used a flatter structure with different field names. The `FindingAdapter` in `models.py` converts v1 findings to the canonical v2 shape automatically. If you encounter findings without `effort` or `risk` fields, they are v1 and will be migrated on read. No action needed from consumers — the adapter handles it transparently.
